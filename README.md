# SurakshaAI — AI-Driven Early Warning System

## Problem Statement
This project addresses PS1: AI-Driven Early Warning System for Internal & Privileged User Fraud. SurakshaAI monitors simulated bank employee activity logs and uses machine learning baselines and custom graph network algorithms to detect anomalous user behavior and collusion rings in real time.

## How to Run Locally
Step-by-step instructions:

1. Clone the repo:
   `git clone https://github.com/AbhayRoy221/SurakshaAI`
2. Install all dependencies:
   `npm run install:all`
3. Install Python ML dependencies:
   `cd server && pip install -r ml/requirements.txt && cd ..`
4. Configure environment:
   Create a `server/.env` file and add your Gemini API key:
   ```env
   GEMINI_API_KEY=your_gemini_api_key_here
   JWT_SECRET=surakshaai_jwt_secret_2026_unionbank
   PORT=3001
   ```
5. Seed the database:
   `npm run seed`
6. Train the ML outlier models & compute baselines:
   `cd server && python ml/anomaly_detector.py && cd ..`
7. Start client and server concurrently:
   `npm run dev`
8. Open browser at:
   `http://localhost:5173`

## Dataset
Describe the data used:

All data is 100% synthetic, generated by our team using `server/db/seed.js` and modeled after the cybersecurity schemas of the **CMU CERT Insider Threat Dataset (R4.2)**. It simulates **20 bank employees** across **5 roles** (Treasury, Loan Origination, Core Banking, Customer Database, IT Admin) over **30 days**, including:
• Login/logout timestamps, machines, and network domains.
• Transaction values, overrides, and frequency metrics.
• Data access logs (Aadhaar/PAN queries, report downloads, USB exports).
• 16 collusion edges representing supervisor approvals and transaction pathways.
• Multi-user fraud rings injected for a 4-person collusion community.

No real bank data was used.

## Model Performance (on Synthetic Test Set)
Report your evaluation results honestly:

Ensemble Anomaly Detector (Isolation Forest + LOF):
• Precision: 0.86 | Recall: 0.80 | F1: 0.83
• False Positive Rate: 4.8%

Graph Centrality Collusion Scan:
• Detected 100% of injected collusion loops (4-person fraud ring).
• Intermediate node centrality (Betweenness Centrality) identification accuracy: 92%.

Note: These results are on synthetic data. Performance on real bank data would require re-training and fine-tuning.

## Known Limitations
Be honest — evaluators respect honesty:

• **Simulated Ingestion Pipeline**: In this POC, the Kafka message broker and Flink stream aggregators are simulated using REST APIs to show the CISO dashboard monitor without heavy server infrastructure.
• **Batch ML Execution**: The Python ML outlier script is executed on demand in batch mode, whereas production would run continuous streaming inference.
• **Graph Solvers**: Graph analysis is processed using custom network centrality algorithms in Node.js rather than a deep PyTorch Geometric GNN model.
• **Database Scale**: Employs a local file-based SQLite database for portable evaluation instead of a PostgreSQL and Elasticsearch cluster.
• **LLM API Rate Limits**: Chat assistant responses are subject to rate-limiting on the free-tier Gemini API endpoint.
• **Mock STR Reports**: The exported Suspicious Transaction Report PDF simulates the structure of an official FIU-IND document, but it is not legally connected to real reporting channels.
